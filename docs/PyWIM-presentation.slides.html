1. El uso de Python como apoyo al pesaje de vehículos pesados en movimiento (WIM)¶

Muchos accidentes en carreteras son causados directa o indirectamente por vehculos pesados conducidos con sobrepeso. Estos causan daos en el pavimento y tambin sufren ms efectos dinmicos durante las curvas.

Para inhibir el exceso de peso de estos vehculos es necesario fiscalizar estas infracciones y, cuando necesario, aplicar las medidas establecidas por ley, como multas y aprehensiones. Un mtodo que est siendo investigado en muchas partes del mundo es el pesaje en movimiento. Este mtodo tiene como ventajas la economa en espacio fsico y operacin, ya que sus sensores son instalados en la propia carretera y no implica en atrasos en el viaje de los usuarios de la va, pues puede pesar los vehculos pesados transitando en la velocidad directriz de la va.

En este trabajo sern presentados tecnologas tiles para desarrollar un sistema computacional para apoyo al pesaje de vehculos en movimiento. La experiencia para desarrollar este trabajo fue obtenida a travs del proyecto desarrollado en el laboratorio de transportes (LabTrans) de la Universidade Federal de Santa Catarina (UFSC). El objetivo de este trabajo es servir como base inicial para futuros investigadores del tema.

El lenguaje utilizado aqu ser el Python y las libreras principales utilizadas sern: numpy, scipy, pandas, sqlalchemy, statsmodels, numba, scikit-learn, pydaqmx, matplotlib.

2. Descripción del proyecto¶

Un sistema computacional de pesaje de vehculos en movimiento est compuesto, bsicamente, de:

  • Adquisicin de seal de los sensores de peso en la va);
  • Segmentacin de seal (para recortar la seal respectiva al camin medido);
  • Tratamiento de seales;
  • Clculos (velocidad, nmero de ejes, grupos de ejes, distancia entre ejes, peso total, peso por ejes, peso por grupo de ejes, largo);
  • Clasificacin del vehculo;
  • Calibracin;
  • Reconocimiento de matrculas vehiculares;
  • Deteccin de infraccin;

El sistema debe ser rpido y robusto para procesar todas estas informaciones en el menor tiempo posible. Python no es un lenguaje reconocido por tener un alto desempeo, por eso, es necesario utilizar libreras y mtodos para potenciar su capacidad de procesamiento.

Con base en los resultados del pesaje, clasificacin y reconocimiento de la matrcula vehicular es posible saber si el vehculo cometi alguna infraccin y, en caso positivo, es posible vincular la infraccin a la identificacin del vehculo infractor.

3. Adquisición de datos¶

La adquisicin de datos fue hecha a travs de placas de adquisicin DAQmx de la empresa National Instruments (NI). Para comunicar con estas fue utilizada la librera PyDAQmx, un wrap hecho en Python para los controladores del hardware fornecidos por la empresa. Despus de adquirir la seal de los sensores, el sistema la almacena en un buffer circular en memoria que, dentro un proceso paralelo, es analizada en busca de una seal completa de un vehculo (segmento). 6.4 Pesos¶

Para calcular el peso del vehculo sern utilizadas las informaciones de velocidad, la curva de cada eje, . Para los sensores Lineas de Kistler, debe ser seguida la siguiente formula (KistlerInstrumente, 2004): $W = ( V / L_s ) * A * C$, donde W es la variable de peso, V es la velocidad, $L_s$ es el ancho del sensor, A es la integral de la curva y C es una constante de calibracin. Para otros tipos de sensores, la frmula es similar. Para sensores del tipo piezoelctrico polmero y cermicos es necesario considerar un mtodo para correccin de los resultados debido a la sensibilidad a la temperatura (Burnos, 2008), (Gajda et al., 2012). Para los datos de ejemplo, sern calculados los pesos sobre los ejes y el peso bruto total utilizando como parmetro: el ancho del sensor con el valor de 0.53 metros y la constante de calibracin igual a 1 para todos los sensores.

7. Clasificación de vehículos¶

Aqu ser presentado un mtodo para clasificacin vehicular basado en los trabajos de (vanBoxel and vanLieshout, 2003) y (Oommen and Loke, 1997)

En este mtodo, es utilizado un conjunto de layouts de referencias, definido por un conjunto de smbolos, que representa el diseo del vehculo, como puede ser visto en la figura abajo (vanBoxel and vanLieshout, 2003).

Para clasificar el vehculo, el sistema crea un layout para el vehculo medido, lo compara con layouts de referencias y clasifica el vehculo que con el layout de referencia que resulta ms prximo.

Este mtodo presenta bajo desempeo en el lenguaje Python. Para solucionar esto, fue utilizada la librera numba, llegando a ser cerca de 100 veces ms rpido. Fue necesria una adaptacin en el algoritmo donde, ante de hacer las comparaciones, el layout del veculo y el layout de la clase de referencia son convertidos en nmeros, as la funcin de comparacin puede ser marcada para ser compilada en modo nopython. Cuanto ms cerca de 0 ms cerca el layout del vehculo est del layout de referencia.

def score_95_calc(metric_score, y, y_pred):
    if y.shape[0] < 1:
        print('size calc 0')
        return 0.0

    y_true = np.array([True] * y.shape[0])
    lb, ub = y - y * 0.05, y + y * 0.05

    y_pred_95 = (lb < y_pred) == (y_pred < ub)
    y_pred_95 = y_pred_95 == True

    return metric_score(y_true, y_pred_95)


def score_95_base(metric_score, estimator, X_test, y_test):
    if y_test.shape[0] < 1:
        print('size base 0')
        return 0.0
    y_pred = estimator.predict(X_test)
    return score_95_calc(metric_score, y_test, y_pred)

El reconocimiento de matrculas vehiculares puede ser realizado a travs de una cmara ALPR. Tambin es posible hacer este procesamiento separado, sin utilizar la cmara ALPR. Un material muy interesantes sobre el tema es el trabajo (Sajjad, 2010) que provee informaciones y ejemplos de cmo hacer el reconocimiento de matrculas vehiculares utilizando el lenguaje Python junto con la librera OpenCV. La matrcula vehicular es muy importante para los sistemas de pesaje de vehculos pesados pues con esta informacin es posible penalizar los infractores, principalmente los que huyen despus de recibir el aviso de detencin en las estaciones de control de pesaje.

10. Conclusión¶

Este es un trabajo inicial con propsito educativo de cmo crear un sistema computacional de apoyo al pesaje de vehculos pesados en movimiento. Los prximos pasos para evolucionar este trabajo es ampliar los temas de:

  • Adquisicin de datos, desde la configuracin elctrica, hasta los parmetros de configuracin de la misma;
  • Reconocimiento automtico de matrculas vehiculares, utilizando alguna librera y desarrollando algunos ejemplos;
  • Diferencias entre algoritmos para clculo de pesaje entre sensores del tipo piezoeltrictos cuarzo, cermico y polmero.
